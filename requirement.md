# Project Requirements / Yêu Cầu Dự Án
**Project Title:** Implementing Security Features in an Auto Maintenance Application  
**Tên đề tài:** Triển khai tính năng bảo mật trong xây dựng ứng dụng bảo dưỡng ô tô  

---

## 1. Objectives / Mục tiêu
**EN:**  
- Analyze and design a simple auto maintenance application with Oracle security features.  
- Study Oracle security techniques (encryption, authentication, access control).  
- Implement a functional and secure prototype application.  
- Train self-learning, research, and teamwork skills.  
- Improve responsibility, creativity, and persistence during the project.  

**VN:**  
- Khảo sát, phân tích và thiết kế ứng dụng bảo dưỡng ô tô có áp dụng các tính năng bảo mật của Oracle.  
- Tìm hiểu các kỹ thuật bảo mật của Oracle (mã hóa, định danh, kiểm soát truy cập).  
- Cài đặt và triển khai ứng dụng mẫu có tính bảo mật cao.  
- Rèn luyện kỹ năng tự học, tự nghiên cứu và phối hợp nhóm.  
- Nâng cao tinh thần trách nhiệm, sáng tạo và kiên nhẫn trong quá trình làm đồ án.

---

## 2. Scope / Phạm vi
**EN:**  
- The system manages service centers, customers, and service providers.  
- Secure modules: authentication, authorization, auditing, QR-based verification, digital signature.  
- Use Oracle DB, implement VPD/OLS for fine-grained access control.  

**VN:**  
- Hệ thống quản lý địa điểm, khách hàng, đơn vị cung cấp dịch vụ.  
- Các mô-đun bảo mật: đăng nhập, phân quyền, kiểm toán, xác thực bằng mã QR, ký số hóa đơn.  
- Sử dụng Oracle Database, áp dụng VPD/OLS để điều khiển truy cập chi tiết.

---

## 3. Functional Requirements / Yêu cầu chức năng
| # | English | Tiếng Việt |
|---|----------|------------|
| 1 | Manage locations, service types, customers, and suppliers | Quản lý địa điểm, loại hình dịch vụ, khách hàng và nhà cung cấp |
| 2 | Create and manage service orders | Tạo và quản lý đơn đặt dịch vụ |
| 3 | User registration and authentication | Đăng ký và xác thực người dùng |
| 4 | Access control via roles (Admin, Employee, Customer) | Điều khiển truy cập theo vai trò (Quản trị, Nhân viên, Khách hàng) |
| 5 | Data encryption for sensitive fields | Mã hóa dữ liệu nhạy cảm |
| 6 | Apply VPD or OLS for row-level security | Áp dụng VPD hoặc OLS để bảo vệ dữ liệu theo hàng |
| 7 | Implement FGA and standard audit logging | Áp dụng FGA và Standard Audit để kiểm toán hệ thống |
| 8 | Generate invoices with digital signatures | Lập hóa đơn có ký số |
| 9 | Generate QR code for service lookup | Sinh mã QR để tra cứu dịch vụ |
|10 | Backup and restore database | Sao lưu và phục hồi cơ sở dữ liệu |

---

## 4. Non-Functional Requirements / Yêu cầu phi chức năng
**EN:**  
- Secure coding and Oracle best practices.  
- Compatible with Oracle Linux, Oracle DB 18c+.  
- UI designed for simplicity and clarity.  

**VN:**  
- Tuân thủ nguyên tắc lập trình an toàn và tiêu chuẩn Oracle.  
- Tương thích với Oracle Linux, Oracle Database 18c trở lên.  
- Giao diện thân thiện, dễ sử dụng.

---

- Programming: C# or Java

**VN:**

- Ngôn ngữ: C# hoặc Java  
- Hệ điều hành: Oracle Linux
